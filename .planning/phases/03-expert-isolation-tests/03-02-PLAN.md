  <action>
Create `runpod/test4-inference.py` -- a post-training inference validation script adapted from `runpod/test3-inference.py` for single-expert LoRA checkpoints. Then run it on the pod.

**Script structure:**

1. **Discover checkpoints** -- list what's in test4-low-only and test5-high-only output dirs. Find the final epoch checkpoint for each.

2. **Load T5 and encode prompts** -- same pattern as test3-inference.py. Use `WanInferencePipeline._fix_t5_embed_tokens()`. Encode 2 prompts:
   - "Medium shot, Holly Golightly walks up an indoor staircase, looking back over her shoulder, morning light." (same as training)
   - "Close up, Holly Golightly sits at a vanity mirror, adjusting a pearl necklace, soft warm lighting." (unseen prompt)

3. **Load both expert base models** -- `WanTransformer3DModel.from_single_file()` with explicit `config=` parameter (diffusers bug workaround). Build WanPipeline with shift=5.0 scheduler, boundary_ratio=0.6 (inference boundary, NOT 0.875).

4. **Test A: Base model (no LoRA)** -- generate 2 videos as baseline. Save to `/workspace/outputs/test4-low-only/inference_test/base_*.mp4`.

5. **Test B: Low-noise LoRA only** -- load the low-noise checkpoint via `pipeline.load_lora_weights()`. The low-noise LoRA has `transformer_2.` prefix keys, so it applies to the low-noise expert (transformer_2 in the pipeline). Generate 2 videos. Save as `low_lora_*.mp4`. Then unload.

6. **Test C: High-noise LoRA only** -- load the high-noise checkpoint. The high-noise LoRA has `transformer.` prefix keys, so it applies to the high-noise expert (transformer in the pipeline). Generate 2 videos. Save as `high_lora_*.mp4`. Then unload.

7. **Automated quality checks** -- for each generated video, run 3 numpy checks:
   - Not black: mean pixel value > 10 (catches failed generation)
   - Has motion: mean frame-to-frame diff > 1.0 (catches frozen output)
   - Has variance: pixel variance > 100 (catches flat solid color)
   Print PASS/FAIL for each check with numeric values.

8. **Loss trend analysis** -- pull loss data from W&B or parse console output. Report whether loss EMA trended downward over 5 epochs for both runs. A generally downward trend is sufficient (not strict monotonic per CONTEXT.md).

**Inference settings (validated in Phase 1):**
- shift=5.0, boundary=0.6 (inference, NOT training boundary)
- 30 steps, CFG 4.0
- 480x832, 17 frames
- seed=42 for reproducibility

**Key insight about single-expert LoRA inference:**
When only one expert has a LoRA, the other expert runs with base weights. This is correct and expected -- the LoRA only trained one expert, so only one expert should be modified. The dual-expert pipeline with one LoRA is the correct way to evaluate these checkpoints.

python /workspace/dimljus/runpod/test4-inference.py
